# 天眼查企业数据管理

## 数据概况
- 数据库：`~/.openclaw/plugin-skills/tianyancha/tianyancha.db`
- 主表：`enterprise_detail`（45列，200,001条）
- 数据治理字段：region（区县）、street（街道）、building（大楼宇）、small_building（小楼宇）、account_manager（客户经理）
- 标注范围：仅限盐南高新区 + 经开区（其他区县不计入统计）

## 标准导入流程（每次导入新数据必须执行）

### 第一步：导入 Excel
```bash
cd ~/.openclaw/plugin-skills/tianyancha
python3 scripts/import_excel.py /path/to/【天眼查】高级搜索YYYYMMDD(...).xlsx
```
**导入规则：**
- 按【统一社会信用代码】去重，重复则覆盖现有数据行
- 自动从文件名提取采集日期（例：`20260522` → `2026-05-22`）
- 自动跳过无统一社会信用代码的无效行
- 导入完成后**自动触发数据治理标注**（无需手动再跑 annotate_all）

### 第二步：确认标注结果
标注完成后自动输出各字段填充统计。如需手动触发标注：
```bash
python3 scripts/annotate_all.py
```

### 第三步：查询验证
```bash
python3 scripts/query.py                    # 全量统计
python3 scripts/query.py --region 盐南高新区  # 按区域筛选
```

## 当月新增企业导入流程

从当月新增企业 Excel 导入数据到主库，自动完成去重、区县过滤和数据治理标注。

### 数据源
- 输入文件（默认位置）：`~/.openclaw/plugin-skills/tianyancha-new-enterprise/当月新增企业.xlsx`
- （注：该目录已整合进 tianyancha skill，`tianyancha-new-enterprise` 独立 skill 已废弃）
- 数据格式：Excel，首行为表头，关键列：S列=统一社会信用代码，L列=所属区县，AA列=注册地址

### 全程操作

**Step 1：导入数据到主库**
```bash
cd ~/.openclaw/plugin-skills/tianyancha
python3 scripts/import_excel.py /path/to/当月新增企业.xlsx
```
导入完成后**自动触发数据治理标注**。

**Step 2：导出清洗后名单（Excel）**
```bash
cd ~/.openclaw/plugin-skills/tianyancha
python3 scripts/query.py --output excel --year 2026
```
可按需加 `--region` / `--street` / `--building` 筛选。

**Step 3：生成汇总报告（PDF）**
```bash
cd ~/.openclaw/plugin-skills/tianyancha
python3 scripts/query_summary_pdf.py
```
生成按区县/街道/大楼宇汇总的 PDF 报告。

### 强制规则（同全量流程）
- Excel：必须通过 `scripts/query.py --output excel` 生成，禁止 openpyxl 直接拼
- PDF：必须通过 `scripts/query_summary_pdf.py` 生成
- 报告文字：必须遵从脚本实际输出，禁止 LLM 补充虚假字段描述

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 清空所有 region/street/building/small_building | 重新标注，覆盖旧数据 |

| 8 | 小楼宇匹配 | 特殊规则 + 通用楼号规则 |

### Excel 企业清单导出（query.py --output excel）

**⚠️ 零容忍错误：禁止生成 7 列 Excel**

**❌ 错误格式（7列）：**
区县 | 街道 | 大楼宇 | 小楼宇 | 注册时间 | 企业名称 | 联系号码
→ 这是旧格式/错误格式，缺少后36列字段
→ 触发场景：自己写 Python/openpyxl 拼 Excel、绕过 query.py
→ **任何时候都不能生成 7 列 Excel**

**✅ 正确格式（44列）：**
区县 → 街道 → 大楼宇 → 小楼宇 → 注册时间 → 企业名称 → 联系号码 → 统一社会信用代码 → 登记状态 → 法定代表人 → 注册资本 → 省份 → 城市 → 区县（原始） → **注册地址** → 更多电话 → ... → 入库时间
→ **必须通过 query.py --output excel 生成**
→ 44列 = 7固定列序 + 37其余字段（含新增的注册地址）

#### 文件命名规则
```
{区县}_{街道}_{大楼宇}_{年份}年新注册_{是否排除个体}_{数量}条.xlsx
```
示例：
- `盐南高新区_黄海街道_2026年新注册_非个体_128条.xlsx`
- `金融城_全量企业清单_2127条.xlsx`
- `步凤镇_2026年新注册_非个体_45条.xlsx`

#### 列序固定（顺序不可调）
| # | 列名 | 数据库字段 |
|---|------|-----------|
| 1 | 区县 | region |
| 2 | 街道 | street |
| 3 | 大楼宇 | building |
| 4 | 小楼宇 | small_building |
| 5 | 注册时间 | establishment_date |
| 6 | 企业名称 | enterprise_name |
| 7 | 联系号码 | mobile_phone |
| 8+ | 其余字段 | 统一社会信用代码、登记状态、法定代表人……至来源文件（按数据库原有字段顺序追加） |

#### 排序规则
区县 → 街道 → 大楼宇 → **注册时间倒序（最新注册排前面）**，小楼宇列正常展示但不参与排序

### 四、小楼宇匹配规则

#### 通用规则
- 数字+号楼 → 大楼宇名+楼号
- 数字+幢 → 大楼宇名+楼号（数字+幢后面4位不跳过）
- 数字+#楼 → 大楼宇名+楼号
- **排除**：数字+号楼+4位纯数字 → 房间号，跳过
