## 四、小楼宇匹配规则

### 通用匹配规则（适用于所有大楼宇）

| 地址格式示例 | 小楼宇结果 | 规则说明 |
|-------------|-----------|---------|
| 金融城6号楼 | 金融城6号楼 | 数字+号楼 |
| 金融城5幢1604室 | 金融城5号楼 | 数字+幢 → 楼 |
| 金融城6#2201室 | 金融城6号楼 | 数字+#楼 → 楼 |
| 金融城六号楼701 | 金融城6号楼 | 中文数字转换 |
| 大数据B12二层 | 大数据B12号楼 | 字母+数字+楼 |
| 大数据A楼 | 大数据A号楼 | 单独字母+楼 |
| 涌鑫中心1号637室 | 涌鑫经贸中心1号楼 | 子名称统一 |
| 金融城1号楼1604室 | 金融城1号楼 | 4位数字=房间号，跳过 |
| 金融城1号楼1604A | 金融城1号楼 | 4位数字+字母→不跳过 |

### 排除规则
- 数字+号楼后面紧跟**4位纯数字** → 判定为房间号，跳过（如 `金融城1号楼1604室` → `1604` 是房间号）
- 但 `金融城1号楼1604A` → 保留楼号（4位数字后有字母）
- **数字+幢后面4位不跳过**（楼号+房间号都要保留）

## 五、匹配优先级链

```
P1: 大楼宇命中已知列表 → 直接采用大楼宇对应区域+街道
P2: 街道匹配已知映射 → 采用映射表对应区域
P3: 街道有值但区县为空 → 根据街道补全区县
P4: 大楼宇已匹配但region/street为空 → 根据大楼宇补全
P5: county列=盐城经济技术开发区 → 经开区
P6: registered_address含盐南/城南且county为亭湖/盐都 → 盐南高新区
```

## 六、执行流程

```
Step 1: 清空 region, street, building, small_building
Step 2: 区县匹配（county + address关键词）
Step 3: 街道匹配（address关键词）
Step 4: 街道补匹配区县（街道→区域映射）
Step 5: 大楼宇匹配（address关键词）
Step 6: 大楼宇补匹配region/street（大楼宇固定映射）
Step 7: 同一大楼宇多街道修正（按规则统一）
Step 8: 小楼宇匹配（特殊规则+通用规则）
```
